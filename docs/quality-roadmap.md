# 同人誌として頒布できる品質に上げるためのロードマップ

試作（4ページ）を「世に出せる」水準へ引き上げるための具体的な改善項目を、
インパクトと着手しやすさの順にまとめる。各項目は本リポジトリの該当箇所を併記する。

## 0. 前提：どこを直すと効くか

- 画像生成モデルには日本語文字を描かせず、写植は後段レンダリングで行う方針は維持する
  （`AGENTS.md`）。台詞のにじみ・誤字は写植側で根治する。
- 「世に出せない」と感じる主因は概ね次の3つ。優先度もこの順。
  1. **キャラの描き分け・同一性**（莉嘉が出ない／毎コマ顔が変わる）
  2. **写植の自然さ**（縦書きの体裁、フォント、吹き出し追従）
  3. **誌面としての体裁**（解像度・塗り足し・表紙・奥付）

---

## 1. キャラの同一性・描き分け（最優先）

### 1-1. trigger prompt を素モデルが解釈できるタグにする（本PRで対応）
- 日本語名（例:「城ヶ崎莉嘉」）の trigger は素モデルが解釈できず人物が出ない。
  英字 booru タグ（`jougasaki rika, idolmaster cinderella girls`）へ統一する。
- 本PRで「英字を含まない trigger は弱trigger」とみなし、知識DBのタグで自動補完する
  ようにした（`backend/app/story.py` の `_is_weak_trigger` / `build_characters_from_knowledge`）。
- **既存プロジェクトは再適用するか、制作タブのキャラ編集で trigger を手動更新すると直る。**

### 1-2. 複数キャラのコマは領域分離（regional prompting）を入れる
- いまは1コマ内の複数キャラを通常promptで近接配置しているだけ（`prompt_composer.py`
  の `compose_panel_prompts`）。素モデルでは姉妹のように似た2人が1人に融合し、
  片方（＝より強いタグ side）に寄る。**これが「莉嘉が美嘉になる」根本原因の本命。**
- 対策（効果順）:
  - ComfyUI の **regional conditioning / Attention Couple / GLIGEN** ワークフローを用意し、
    `character_layout` の position を実際の領域マスクへ写像する。
  - それまでの暫定策として、**人物が2人以上のコマは避け、1人ずつのコマに割る** よう
    台本段階の指示を強める（`story.py` の `stage_instruction_text("script")`）。

### 1-3. キャラ LoRA / IPAdapter で同一性を固定
- trigger だけでは毎コマ微妙に変わる。**キャラ別 LoRA** か **参照画像＋IPAdapter** を
  各キャラに紐付ける（`Character.lora_*` / `reference_image_asset` は既に対応済み）。
- キャラ1人につき「正面・表情差分」の参照を1枚決め、全コマで同じ参照を使う運用にする。

### 1-4. seed 運用
- キャラ確定後は、顔が良かったコマの seed を控え、同キャラのリテイクで再利用する。
  生成結果に prompt/seed/backend/status を残す方針（`AGENTS.md`）を活かす。

---

## 2. 写植（縦書き日本語）の自然さ

### 2-1. 句読点の位置（指摘の件・現状の確認）
- 縦書きの `、` `。` は **セル右上**へ寄せる実装が既に入っている
  （`backend/app/typeset.py` の `_draw_glyph(..., corner="tr")` / 227–234行）。
- もし出力で左寄せに見える場合は、(a) 古いビルドの結果、(b) 画像生成モデルが文字を
  焼き込んでしまっている（写植経路を通っていない）、のどちらか。`draw_layout` 経由で
  描かれているかをまず確認する。

### 2-2. フォント
- 商業頒布では**ライセンスが頒布可のフォント**を使う（源ノ角ゴシック/明朝、IPAex、
  やわ明、しっぽくなど）。本文＝明朝系、強調・SFX＝太ゴ/手書き系の2系統を用意。
- フォント指定は `fonts.py` 経由。台詞用と擬音用でフォントを分けられるようにする。

### 2-3. 吹き出し追従・組版の詰め
- 既に禁則処理・縦中横・小書き仮名・長音回転は実装済み（`typeset.py`）。残る伸びしろ:
  - **ルビ（ふりがな）** 対応。
  - 行間・字間の微調整（現状は等間隔）。
  - 吹き出し形状に合わせた**多角形マスク内フィッティング**（楕円/トゲ付き）。
  - 感情表現の**フォントサイズ強弱**（叫び＝大、心の声＝小・別フォント）。

### 2-4. SFX（描き文字）
- 擬音は台詞と分離する設計が既にある（`review_script` が擬音を sfx へ移す）。
  手書き風・インパクト系のスタイルを増やし、コマ外にもはみ出せるようにする。

---

## 3. 誌面・作画の体裁

- **解像度**: 表示確認用の小さい生成ではなく、最終は **印刷用 350dpi** 相当へ。
  生成→**ハイレゾ/アップスケール**（hires fix / ControlNet tile）を後段に追加。
- **モノクロ同人の場合**: グレースケール化＋**トーン（網点）**処理、白黒2値の線画調整。
  カラーで出すなら全ページのトーン・彩度を揃える。
- **コマ割りの変化**: いまは固定グリッド寄り（`layout_engine.py`）。見開き・断ち切り・
  斜めコマなど緩急を付けると「誌面」らしくなる。
- **塗り足し（3mm）・トンボ・ノド側の余白**を出力に持たせる（印刷入稿要件）。

---

## 4. ストーリー/演出の質

- 1コマ台詞は短く、ページ末に引きを作る方針は既に知識DB側にある
  （`data/knowledge/.../required.json` の creative_policy）。さらに:
  - **キャラの口調**を `speech_style` で固定し、台詞の人物差を出す。
  - **表情・カメラの指定語彙**を増やし、棒立ち・同一構図の連続を避ける。

---

## 5. 入稿・頒布フロー

- **表紙・裏表紙・奥付**（サークル名・発行日・連絡先・無断転載禁止）を生成対象に加える。
- 出力は **PDF/X or 入稿用 PNG（CMYK・350dpi・塗り足し込み）**。`exports` 系を印刷用と
  Web公開用で分ける。
- 公開前チェック: 権利表記、年齢区分、モザイク要否（全年齢なら不要）を確認。

---

## 推奨着手順（次の3歩）

1. **1-1（本PRで対応済み）＋1-2の暫定策**：複数人コマを1人コマへ割り、莉嘉を確実に出す。
2. **1-2本命**：ComfyUI に regional/Attention Couple ワークフローを追加し、領域分離を実装。
3. **2-2/3**：頒布可フォント導入とルビ・吹き出しフィッティングで写植を商業水準へ。
